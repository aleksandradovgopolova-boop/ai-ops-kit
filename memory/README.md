# Repository Memory

Хранит только устойчивые знания, полезные будущим задачам.

- `decisions/` — ADR и важные product/technical decisions;
- `patterns/` — проверенные reusable patterns;
- `incidents/` — выводы из инцидентов;
- `known-issues/` — подтверждённые ограничения;
- `lessons-learned/` — повторяемые уроки.

Каждая запись содержит источник, owner, дату проверки и условие устаревания.

## Когда память пополняется (контракт)

Workflow ENGINEERING, PRODUCT и RESEARCH завершаются стадией `memory-capture` /
`knowledge-update` (owner — repository-memory-curator): опубликовать запись в memory/
**или явно отказаться с причиной в TaskState**. Молчаливый пропуск не является
выполнением стадии. Для incident-resolution запись в `incidents/` обязательна —
отказ не предусмотрен.

Типичные причины обоснованного отказа: изменение не содержит переиспользуемого
знания; урок уже зафиксирован (дать ссылку); знание временное и устареет до
следующей задачи.
