# Measurement Baseline

- ни одна пользовательская функция не выходит без инструментации (TrackingPlan приложен и проверен);
- у каждой функции есть primary metric и хотя бы один guardrail;
- единый стандарт именования событий (object_action, snake_case), без PII в именах и значениях без обоснования;
- метрики определяются один раз в MetricCatalog; в дашбордах и планах — ссылки, не переопределения;
- baseline фиксируется до релиза, target — явный;
- инструментация проверяется на конкретной ревизии (событие реально доходит до destination);
- эффект интерпретируется с учётом ограничений (сегменты, сезонность, объём выборки);
- «измеримость» — часть Definition of Done, а не пост-фактум.
- для AI-фич (LLM/агент в продукте) продуктовые метрики дополняются eval'ами качества модели/промпта (см. rules/ai/EvalPolicy.md и AIFeatureEvalPlan).
