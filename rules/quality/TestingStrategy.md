# Testing Strategy

Тестировать поведение на наиболее дешёвом подходящем уровне:

- unit — локальная логика;
- component — граница компонента;
- integration/contract — взаимодействия;
- e2e — критичные пользовательские пути.

Каждый дефект по возможности получает regression test. Высокий процент coverage не заменяет проверку рисков и бизнес-сценариев.
