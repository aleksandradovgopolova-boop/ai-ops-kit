# Cost and Token Policy

- Сначала исследовать индекс и локальный контекст, затем расширять поиск.
- Использовать дорогие reasoning-модели только для задач, где это снижает риск.
- Не дублировать один анализ несколькими агентами без цели независимой проверки.
- Сохранять компактные context packs и handoff вместо повторного чтения.
- Оптимизировать общую стоимость результата, а не цену отдельного вызова.
