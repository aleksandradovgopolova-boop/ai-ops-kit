# Parallel Work

- Один write-владелец на файл.
- Shared contracts замораживаются до начала параллельной реализации.
- Read-only роли могут работать параллельно.
- Каждая write-задача имеет явный список разрешённых файлов.
- Результаты передаются через TaskHandoff.
- После объединения обязательны общие integration/regression checks.
