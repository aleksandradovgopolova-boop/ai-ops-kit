# Context Management

## Приоритет чтения

1. Локальные правила ближайшей директории.
2. TaskBrief, TaskContext и TaskState.
3. Канонические продуктовые и архитектурные документы.
4. Код, контракты и тесты.
5. Memory и история решений.

## Ограничения

- читать только релевантную область;
- не загружать весь репозиторий без причины;
- резюмировать контекст ссылками на пути;
- помечать дату и источник факта;
- исключать устаревшие материалы или явно маркировать их.
