# FreshnessPolicy — классы устаревания знаний (v2.9)

Идея из team-os-toolkit (init-team-brain): доверие к документу зависит от того, как
быстро он устаревает и когда его последний раз проверяли. Документ без отметки
свежести — это утверждение без срока годности.

## Единый термин

Класс свежести задаётся полем **`stability`** во frontmatter документа (термин уже
используется в ките — второго понятия `tier` не заводим).

```yaml
---
stability: volatile        # stable | evolving | volatile
reviewed_at: 2026-07-13    # дата последней проверки человеком (YYYY-MM-DD)
expires_after_days: 14     # опционально; по умолчанию — из класса ниже
owner: product-team        # опционально
sources:                   # опционально: где живой источник истины
  - src/product/types.ts
---
```

## Классы и сроки по умолчанию

| Класс | Что это | Срок по умолчанию | Примеры |
|---|---|---|---|
| `stable` | Меняется редко; основа | не истекает | роли ЛПР, базовая терминология, архитектурные принципы, нормативный процесс |
| `evolving` | Меняется по мере развития продукта | 90 дней | карта продукта, дизайн-система, процессы команды, стейкхолдеры, перечень ИС |
| `volatile` | Живое состояние | 14 дней | статус инициатив, приоритеты недели, активный backlog, метрики, текущие риски |

## Правило доверия

- Документ `volatile`/`evolving`, у которого `reviewed_at + срок < сегодня`, считается
  **протухшим**: агент не имеет права использовать его как актуальный источник и должен
  сослаться на живой source of truth или запросить обновление.
- `validate_freshness.py` помечает протухшие документы (advisory WARN, не блокирует).

## Что НЕ хранить в репозитории как «истину»

Живое состояние (статусы задач, свежие метрики, доступность сервисов) не дублируется в
markdown — репозиторий проиграет Jira/BI/мониторингу и при этом будет уверенно лежать
устаревшим. Репозиторий хранит **где** источник истины, **как** его получить, **кто**
владелец и **что** означают поля. Единственное исключение — датированный **`now.md`**:
он явно снимок на дату, а не живая истина (см. `context/now.md`).

## Для child-репозиториев

Размечать `stability` в первую очередь у контекстных документов
(`context/product/*`, `context/team/*`) и держать `now.md` как еженедельный снимок.
