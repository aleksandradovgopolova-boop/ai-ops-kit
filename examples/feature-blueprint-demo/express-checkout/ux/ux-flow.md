# UX Flow

## User goal
Оформить повторный заказ за минимум действий.
## Entry points
Корзина -> кнопка «Экспресс-чекаут» (только у повторных с адресом).
## Flow (шаги пользователя; mermaid/текст)
Корзина -> Экран подтверждения (адрес+оплата+состав) -> Заказ оформлен.
## Screen list
| Screen | Purpose | Key components (из дизайн-системы) | Navigation |
|---|---|---|---|
| ExpressConfirm | Подтверждение заказа | AddressCard, PaymentSelector, PrimaryButton | Корзина -> Confirm -> Success |
## User journey (эмоции/фрикции по шагам)
Ожидание скорости; фрикция — неактуальный адрес (inline-редактирование).
## Wireframes (описание или ссылки)
Один экран: состав (свёрнут), адрес, оплата, итог, CTA.
## Copy (ключевые тексты, тон)
CTA «Оформить за 1 шаг»; ошибки — с действием восстановления.
## Open questions
Показывать ли промокод-поле (риск отвлечения).
