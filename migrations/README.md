# migrations/ — миграции между версиями пакета

Каждая миграция — папка `<from>-to-<to>/` с `up.py` (применение) и `down.py` (rollback).
Цепочка объявляется в `manifest/ai-ops-manifest.yaml -> package_migrations.chain`
и применяется командой `ai-ops migrate` (детерминированно, по порядку).

Правила:
- `up.py`/`down.py` принимают один аргумент — корень child-репозитория; exit 0 = успех;
- миграция идемпотентна (повторный запуск не ломает состояние);
- необратимая миграция обязана иметь `down.py`, который честно завершается ошибкой
  с объяснением, и помечается в README миграции как irreversible;
- миграции не читают и не пишут секреты.

Шаблон — `_template/`.
