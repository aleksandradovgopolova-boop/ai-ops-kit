# Постоянный контекст

Заполняется командой и является источником проверенной информации, а не складом всех документов.

- `product/` — продукт, аудитории, бизнес-правила и метрики;
- `system/` — архитектура, интеграции, данные и окружения;
- `team/` — ownership, процесс и glossary.

Каждый файл должен содержать owner, дату последней проверки и ссылки на первичные источники.
