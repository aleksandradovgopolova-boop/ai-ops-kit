# ProductStatus.md — что реально готово и живёт сейчас

**Одна точка правды о фактическом состоянии продукта.** Это НЕ «как должно быть»
(для этого — нормативные JTBD/USE_CASES/MVP_SCOPE/ADR), а «что уже построено, где
работает и что отложено». Каждая новая сессия читает этот файл **первым** (см.
`ai-session-start`), и он обновляется на **каждом PR, меняющем готовность** (см.
`rules/core/ProductStatusPolicy.md`).

Это снимок: он верен на `reviewed_at` из frontmatter. Устаревшую строку исправляет тот
PR, который меняет реальность, — а не «когда-нибудь потом».

## Что реально готово / живёт в проде

Заполняется в child-репозитории. Формат — факт + где это работает + чем подтверждено:

| Область | Статус | Где живёт / чем подтверждено |
|---|---|---|
| Backend / API | (готово / частично / нет) | (деплой, URL/проект, ADR) |
| Генерация / AI-workflow | (…) | (провайдер, модель, ADR) |
| Хранилище данных | (…) | (СУБД/сервис, ADR) |
| Frontend | (…) | (что именно, mock vs реальные данные) |
| Деплой / окружения | (…) | (платформа, прод/preview) |
| Провайдеры/ключи | (…) | (какой провайдер, ключи из env) |

> Пример строки: «Backend / API — **готово**, задеплоен на Vercel (проект X),
> реальный провайдер DeepSeek; ADR-010/ADR-013.»
