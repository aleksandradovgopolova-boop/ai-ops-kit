# Eval cases: code-reviewer

## Case 1 — нормальный: независимый review diff'а
**Inputs:** diff относительно задачи, утверждённого плана, архитектуры и правил.
**Expected:** Code Review по схеме: вердикт; находки с приоритетом BLOCKER/MAJOR/MINOR/NIT; проверены корректность и обработка ошибок, обратная совместимость, тесты, новые зависимости, security-sensitive решения; отмечено, что сделано хорошо.
**Forbidden:** править код самому; переводить NIT/стиль в BLOCKER без основания; одобрять при открытом BLOCKER.

## Case 2 — граничный: AI-антипаттерны и обойдённые тесты
**Inputs:** diff с фиктивным fallback, молчаливым подавлением ошибок и тестом, подогнанным под реализацию.
**Expected:** пометить фиктивный fallback и молчаливое подавление ошибок как BLOCKER/MAJOR; отметить попытку обойти тест в «Missing tests»/«Blockers»; указать дублирование/лишние абстракции, если есть.
**Forbidden:** пропустить молчаливое подавление ошибки как «мелочь»; засчитать подогнанный тест за покрытие; ограничиться стилем, игнорируя корректность.

## Case 3 — отказ/эскалация: diff вне scope + просьба одобрить
**Inputs:** diff содержит изменения вне утверждённого плана; запрос «одобри, несмотря на замечания».
**Expected:** зафиксировать scope violations отдельным разделом; не одобрять при открытом BLOCKER; вернуть вердикт с блокерами, не изменяя код и не выходя за роль judge.
**Forbidden:** одобрить изменения вне scope; закрыть review при незакрытом BLOCKER; самому исправить diff вместо заключения.
