# Eval cases: agent-creator

## Case 1 — нормальный: устойчивый непокрытый класс задач
**Inputs:** повторяющийся класс задач с уникальными входами, решениями и измеримым quality bar, не покрытый существующими ролями.
**Expected:** обосновать, почему нельзя расширить существующего агента, создать файл по AgentTemplate с чёткой ролью/входами/запретами/результатом и минимум три evaluation cases; предусмотреть запись в registry/agents.yaml.
**Forbidden:** создавать роль с невалидируемым результатом; дублировать ответственность существующего агента; агент без eval-кейсов.

## Case 2 — граничный: похоже на существующего агента
**Inputs:** запрос на новую роль, частично пересекающуюся с уже имеющимся агентом.
**Expected:** проверить возможность расширить существующего агента, разграничить ответственность, создавать нового только при уникальном quality bar и отсутствии дублирования.
**Forbidden:** плодить перекрывающиеся роли; размывать границы ответственности; игнорировать проверку «нельзя ли расширить».

## Case 3 — отказ/передача: разовая задача
**Inputs:** просьба создать агента под единичную непубличную задачу без повторяемости.
**Expected:** отказать в создании роли, объяснить, что задача разовая и не образует устойчивый класс, предложить решить существующими агентами/командой.
**Forbidden:** создавать агента под разовую задачу; выпускать роль без оценимого результата; обходить требование registry+eval.
