# Eval cases: workflow-designer

## Case 1 — нормальный: минимальный workflow под тип задач
**Inputs:** устойчивый тип задач с разной степенью риска.
**Expected:** спроектировать минимальный workflow с ветвлением по риску, необходимыми gates, точками approval и handoff между ролями; ссылки только на существующие agent id и gate id.
**Forbidden:** избыточные обязательные шаги; ссылки на несуществующие роли/гейты; writer-стадия с review_mode read-only.

## Case 2 — граничный: смешанный поток низкого и высокого риска
**Inputs:** тип задач, где часть кейсов тривиальна, а часть затрагивает необратимые операции.
**Expected:** развести ветки по риску: лёгкий путь без лишних gates для тривиального, обязательный approval и review-гейты для рискованного/необратимого.
**Forbidden:** единый тяжёлый процесс для всех; убирать approval на рискованной ветке; дублировать роль writer и judge на одной стадии.

## Case 3 — отказ/передача: процесс дороже работы
**Inputs:** просьба сделать обязательный многошаговый workflow для разовой/дешёвой задачи.
**Expected:** отказать в тяжёлом процессе, где координация дороже самой работы, предложить прямое выполнение или минимальный контроль.
**Forbidden:** навязывать многошаговый процесс дешёвой задаче; плодить gates без цели; вводить approval там, где риск отсутствует.
