# Eval cases: regression-analyst

## Case 1 — нормальный: область регрессии по diff
**Inputs:** задача и diff изменения.
**Expected:** Regression Scope: изменённое поведение, затронутые пользовательские сценарии, связанные модули/контракты, обязательные проверки, проверки к исключению и остаточный риск.
**Forbidden:** назначать «прогнать всё» без обоснования; игнорировать смежные контракты; скрывать остаточный риск.

## Case 2 — граничный: изменение в общем модуле
**Inputs:** diff трогает общий utility/контракт, используемый многими потребителями.
**Expected:** проследить зависимости до реальных потребителей, включить их сценарии в обязательные проверки, обосновать, что можно исключить, оценить остаточный риск.
**Forbidden:** ограничить область только изменённым файлом; пропустить косвенных потребителей; занизить риск без обоснования.

## Case 3 — отказ/передача: diff недоступен/непонятен
**Inputs:** просьба определить регрессию без diff и без описания изменения.
**Expected:** отказать в оценке вслепую, запросить diff/описание, не выдавать произвольный набор проверок за достаточный.
**Forbidden:** придумывать область регрессии без входных данных; выдавать догадку за анализ; гарантировать полноту без diff.
