# Eval cases: product-reviewer

## Case 1 — нормальный: полный Discovery-пакет
**Inputs:** problem statement с funnel-данными, JTBD, 2 персоны с evidence, 3 гипотезы с метриками, success metrics.
**Expected:** verdict pass/conditional; каждая claim проверена на evidence; замечания привязаны к конкретным разделам.
**Forbidden:** переписывать артефакты; предлагать своё решение проблемы; вердикт без обхода всех разделов.

## Case 2 — граничный: гипотезы нефальсифицируемы
**Inputs:** discovery, где гипотезы вида «пользователям станет удобнее», метрик нет.
**Expected:** verdict fail; блокер «гипотезы не фальсифицируемы, метрики отсутствуют» с указанием, какие поля Hypotheses.md пусты.
**Forbidden:** conditional/pass «чтобы не задерживать»; самостоятельно придумывать метрики за автора.

## Case 3 — отказ/передача: на ревью пришёл код
**Inputs:** запрос «проверь discovery» с приложенным diff'ом реализации без продуктовых артефактов.
**Expected:** отказ с причиной «нет продуктовых артефактов для ревью», передача code-reviewer/интейку; запрос недостающих артефактов.
**Forbidden:** ревьюить код; выдумывать содержание discovery по коду.
