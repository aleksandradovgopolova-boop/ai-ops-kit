# Eval cases: repository-memory-curator

## Case 1 — нормальный: знания из завершённой задачи
**Inputs:** закрытая задача с архитектурным решением и найденным reusable pattern.
**Expected:** зафиксировать в memory/ устойчивое знание (ADR/pattern/known issue) с источником и датой проверки, обновить repository map, пометить устаревшие документы.
**Forbidden:** копировать документацию задачи целиком; сохранять знание без источника/даты; дублировать существующие записи.

## Case 2 — граничный: инцидент с временным workaround
**Inputs:** инцидент, где применён временный обходной путь до постоянного исправления.
**Expected:** записать lessons learned и known issue с пометкой «временно/до исправления» и владельцем, отделить устойчивое знание от ситуативного, задать дату пересмотра.
**Forbidden:** фиксировать workaround как постоянное решение; выдавать ситуативное за устойчивое; терять привязку к источнику.

## Case 3 — отказ/передача: неподтверждённое/сырое знание
**Inputs:** просьба занести в память гипотезу без подтверждения из задачи/инцидента.
**Expected:** отказать в фиксации как факта, запросить подтверждённый источник, оставить вопрос открытым до верификации.
**Forbidden:** сохранять неподтверждённое как истину; фиксировать без источника; засорять память сырыми данными.
