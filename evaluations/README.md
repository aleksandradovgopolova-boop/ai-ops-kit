# Evaluations

Агенты и workflows должны проверяться на реальных или синтетических кейсах.

## Для агента

Минимум три случая:

1. нормальный кейс;
2. граничный или конфликтный кейс;
3. кейс, когда агент должен отказаться или передать задачу другой роли.

## Оцениваем

- правильность trigger;
- соблюдение scope;
- качество результата;
- отсутствие выдуманных фактов;
- evidence;
- стоимость и длину процесса.

## Где лежат кейсы и как это проверяется

Кейсы агента — в `evaluations/agents/<agent-id>.md` (id = имя файла агента без
расширения; шаблон — `AgentEvaluationCase.md`). CI-гейт
`validation/validate_agent_evals.py` требует файл кейсов для каждого агента,
**добавленного или изменённого** в diff'е — существующие агенты без кейсов не
блокируются, долг закрывается по мере того, как их трогают.
