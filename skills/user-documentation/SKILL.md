# User Documentation

Документация, которую пользователь понимает с первого прохода, и которая **не врёт**,
потому что скриншоты сняты из настоящего интерфейса и пересниматся при изменении UI.

## Структура (Diátaxis)

Определить тип и не смешивать в одном документе:
- **Tutorial** — провести новичка за руку к первому успеху.
- **How-to** — решить конкретную задачу шагами.
- **Reference** — точное описание (поля, состояния, ограничения).
- **Explanation** — почему так устроено.

Шаблоны кита: `templates/documentation/UserGuide.md` (Diátaxis-компас),
`templates/documentation/FAQ.md`, `WhatsNew.md`.

## Скриншоты — из реального UI, не нарисованные

- Снимать детерминированно через `skills/e2e-browser-testing` (Playwright): на каждый
  документируемый шаг — свой скриншот, на нужном классе устройства из Device matrix
  (`context/product/DesignSystem.md`), а не только desktop.
- Хранить рядом с докой; в markdown — с осмысленным alt-текстом (доступность и поиск).
- **Пересъёмка при изменении UI**: скриншоты привязаны к шагам сценария; при обновлении
  экрана прогон переснимает их — документация не расходится с продуктом (тот же принцип,
  что claims/drift для кода).
- Аннотации (стрелки/выноски) — поверх, отдельным слоем, чтобы пересъёмка не требовала
  перерисовки.

## Что документировать

- Реальный путь к ценности, а не тур по кнопкам.
- Состояния Empty/Loading/Error/Success — как выглядят и что делать.
- Крайние случаи и восстановление после ошибки.
- Язык пользователя, без внутреннего жаргона (сверка с `context/team/Glossary.md`).

## Границы и отказы

- не выдумывать интерфейс: если экрана нет — не описывать «как будто есть»;
- не вставлять устаревший/нарисованный скриншот вместо снятого из текущего UI;
- не смешивать типы Diátaxis в одном документе;
- не документировать данные из запретных категорий (`governance/information-boundaries.md`).

## Атрибуция

Методология структуры — Diátaxis (общественное достояние идей). Захват скриншотов — на
базе нашего `skills/e2e-browser-testing` (Playwright, MIT). Машиночитаемый чек-лист —
`rules/documentation/user-docs.yaml`.
