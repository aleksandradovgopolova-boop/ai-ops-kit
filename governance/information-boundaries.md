# information-boundaries — что можно и что нельзя хранить в репозитории (v2.9)

Идея из team-os-toolkit (init-team-os задаёт вопрос «что НЕ должно попадать в этот
репозиторий?» и фиксирует ответ). Для продуктов, работающих с гос-контекстом,
внутренними документами, ПДн и доступами, эта граница — не формальность.

Дополняет, не заменяет: `security/permission-levels.yaml`, `config/protected-paths.yaml`,
инвариант `secrets_in_repo: forbidden`.

## Категории

| Категория | Правило | Примеры |
|---|---|---|
| **Можно хранить** | В репозитории как есть | терминология, роли, архитектурные принципы, карты процессов, публичные факты |
| **Можно после обезличивания** | Только без прямых идентификаторов | пользовательские сценарии, цитаты из интервью, разборы инцидентов |
| **Только ссылка на source of truth** | Значение не дублировать, хранить указатель + владельца | статусы задач, метрики, состав команды из HR-системы, содержимое тикетов |
| **Запрещено помещать в репозиторий** | Никогда, ни в каком виде | секреты, токены, ключи, пароли, приватные ключи, строки подключения |
| **Запрещено передавать внешним моделям** | Не отправлять provider'ам вне контура | сведения ограниченного доступа, ПДн спецкатегорий, гос-тайна |

## Как применяется

- **Люди**: при добавлении документа свериться с таблицей; сомнение = «только ссылка».
- **Агенты**: перед записью в knowledge/context проверять категорию; персональные и
  ограниченные данные — не материализовать, оставлять указатель на источник истины.
- **Human approval**: помещение чего-либо из «после обезличивания» / «только ссылка»
  требует явного решения человека (см. `rules/core/HumanApproval.md`).
- **Внешние модели**: маршрутизация запросов с данными последней категории — только на
  провайдеров внутри контура (см. `registry/routing-policy.yaml`), никогда на внешние.

## Trust boundary для runtime-агентов

Постоянно работающий агент (см. ROADMAP «Runtime/Robin») — read-mostly: чат-ввод
считается недоверенным, инструкции из чата не меняют конфигурацию и не переносят
данные через границы категорий выше.
