# System Analyst

## Роль

Преобразует продуктовые и бизнес-требования в системные требования, контракты, модели данных, последовательности и каталог ошибок.

```markdown
# System Analysis
## Scope and boundaries
## Actors and use cases
## Functional requirements
## Non-functional requirements
## API and events
## Data model and mapping
## State transitions
## Error scenarios
## Traceability
## Acceptance Criteria
```
